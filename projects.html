StartJapanese!

February 2011

In secondary school I took Japanese for one year, during which we had to learn Hiragana and Katakana. I decided to make an app for my Android phone to aid me and StartJapanese was the result. StartJapanese is really just a basic flashcards app with decks for Hiragana and Katakana included by default. It can radomise decks, test the ones you know least first and offers question and revision modes. There is also the ability to add decks

The decks or sets of cards are stored in a database on the phone, as are results from tests

It is available on the play store here

Tools used: Java, SQL, UI Design

Imageception

August 2012

Imageception was created during Young Rewired State 2012 - a hack week for coders under the age of eighteen. The participants are mentored by adults, previous YRS participants and each other at centres dotted around the country to create something with open data. YRS started in the UK but now has events in other countries

The idea behind Imageception was to create images from other images. The first step was to create other images out of and so I wrote a Python script to take a Flickr tag and download the Creative Commons licensed images with that tag. The bulk of the work is done by another Python script that takes an image (the one you want to make) and a directory full of images (the ones you want it made out of) and creates the image.

The new image is built by going through each pixel of the source image, finding the colour and then finding the image that is most like that colour. This is done by adding up all the red values, all the green values and all the blue values of each image and then dividing each colour value by the number of pixels to get an average colour. The image whose average colour is closest to the one desired is picked and pasted into the final image.

I then created a small website that allowed you to click on one of the individual images (the pixels of the original image) and it would then make that image out of images, hence “Imageception”.

The code is on Github here.

Tools used: Python, Python Image Library, The Flickr API

The North-South Divide

August 2013

The North-South Divide was the project I and a few other people created at my second YRS in 2013. The aim of the project was to try and calculate where in the UK the North-South Divide is, using open data. For those who don’t know the North-South divide is the idea that there is a difference between the North of the United Kingdom and the South. These differences can be cultural or economical, for example the South is often though of as richer than the North and the North is often thought to have worse living conditions. You can read more at Wikipedia

The problem with the divide is that it changes depending on who you talk to; some people thing it cuts through the Midlands, others think it’s just below Manchester; so I thought it would be a good idea to try and analyse some data. We decided to pick crime data, as it already had a latitude and longitude which made our lives easier

I then got to work on trying to calculate where the divide is. To do this I split the country up into a grid and counted the number of data points (eg crimes) in the cell. For each column of the country the program would find at which cell the area above it was the most different to the area below it and drew a line. Joining them up gave us a dividing line. The average of that line was then taken.

We then started adding more data sets. The biggest limitation at this point was that the program did not account for population, so the line was weighted towards big cities where there is a bigger population. Therefore we added a population data set and then others. Unfortunately much of this data only had postcodes as a position, so a member of the team wrote a script to lookup the latitude and longitude of the postcode/

The code is on Github here.

Tools used: Python, HTML, Javascript, JQuery, Google Maps API
